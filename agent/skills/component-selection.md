# 器件选型 Skill

## 目标
帮用户根据功能清单选择合适器件或模块，输出关键参数、接口、电源要求、板级资源占用、SDK适配情况、风险和待确认事项。便于后续硬件规划、框图设计、驱动开发、测试和量产评估。

> 注意：不直接生成 BOM，需要硬件工程师审核。

---

## 适用场景
- 已有功能分析，需要选具体器件。
- 用户只描述功能，如“测距”“显示”“电机驱动”“语音播报”，还没型号。
- 用户给出候选器件，需要比较参数、接口、供电、成本、资料完整性。
- 输出可执行模块清单、接口清单和选型依据。

---

## 输入
- 用户需求及补充说明。
- 功能分析结果（功能清单、SDK能力、缺口、风险）。
- SDK结构（components/、bsp/、boards/、examples/）。
- 板卡资源定义。
- 用户提供候选器件、datasheet或采购链接。
- 可用资料来源：SDK文档、examples、官方datasheet、厂商页、分销商页、开源项目、必要时联网搜索。

---

## 核心原则
1. **先选器件，再考虑SDK适配**  
   优先考虑功能、电气、性能、成本、功耗、尺寸、供货和可靠性。SDK支持只用于评估适配成本。

2. **允许联网搜索**  
   优先官方资料，其次厂商、分销商、开源项目、论坛或店铺。非官方信息标为“待确认”。

3. **只选功能相关器件**  
   与需求无关的器件不加入。必要的保护或隔离等可标为“建议补充”。

4. **每个功能至少一个候选器件**  
   建议关键功能给出 2-3 个候选项，并标注推荐/低成本/高性能/不推荐/待确认。

5. **不确定参数标“待确认”**  
   包括电压、电流、逻辑电平、I2C地址、UART波特率、PWM频率、量程、精度、尺寸、供货状态等。

6. **量产结论谨慎**  
   生命周期、替代料、认证、供应风险、发热、结构可靠性等需人工审核。

7. **遵守 SDK 目录约束**  
   不直接修改 SDK 内部目录，如缺驱动只能标缺口或建议后续新增组件。

## 选型步骤
1. **整理功能**  
   提取输入/输出、传感器、执行器、显示、通信、电源、结构、调试测试功能，同时标出必选/可选/待确认/SDK缺口/资源限制。

2. **建立检索关键词**  
   根据功能、接口、场景、成本、性能生成关键词。例如：
   ```text
   VL53L0X ToF I2C module
   SSD1306 OLED 0.96 I2C
   TB6612 motor driver ESP32 PWM
   DRV8833 motor driver 3.3V logic
   MPU6050 IMU I2C module
   SG90 servo PWM current
   UART TTS voice module
   ```

3. **搜索候选器件**  
优先用户提供资料 → 官方 datasheet/厂商页 → 分销商 → SDK examples → 开源项目 → 论坛/博客。  
SDK支持可加快适配，但不是唯一选择。

4. **提取关键参数**  
- 型号、厂商、模块名  
- 供电、电平、电流、接口、地址/通道  
- 性能参数、尺寸、工作温度  
- 外围要求、资料来源、待确认项  

特殊关注：
- 传感器：量程、精度、刷新率、环境条件  
- 执行器：电压、电流、扭矩、保护、发热  
- 显示：分辨率、控制器、接口、显存  
- 通信：协议、速率、延迟、认证风险  
- 电源：输入范围、输出电流、效率、纹波、热设计

5. **检查板级资源**  
GPIO、I2C、SPI、UART、PWM、ADC、DAC、定时器、中断、电源轨、连接器、安装空间等。  
检查资源冲突、电平不匹配、功耗不足、发热或结构冲突。

6. **评估SDK适配**  
状态：已支持/部分支持/接口简单/需新增驱动/风险高/资料不足  
工作量：低/中/高/很高

7. **比较候选项**  
维度：功能匹配、电气兼容、接口复杂度、资源占用、性能、成本、采购、资料完整性、量产风险  
推荐等级：推荐/可选/待确认/不推荐  
必须说明推荐理由。

## 常见功能器件方向（非资深用户易懂版）
- **LED灯**：板载或外接单色/彩色，注意GPIO、PWM、限流  
- **按键**：板载或轻触开关，注意消抖和上拉/下拉  
- **OLED显示**：SSD1306/SH1106，注意分辨率、I2C地址、显存  
- **测距**：VL53L0X/1X，注意量程、刷新率、环境光、I2C地址  
- **IMU**：MPU6050/ICM42688，注意量程、采样率、安装方向  
- **舵机**：SG90/MG90S，注意PWM、供电、电流、扭矩  
- **电机驱动**：TB6612/DRV8833，注意电流、PWM、保护、编码器  
- **语音**：UART TTS模块，注意协议、波特率、功放、喇叭  
- **WiFi/上位机**：ESP32/内置WiFi/UART，注意协议、延迟、吞吐  
- **摄像头**：OV2640/OV5640，注意接口、帧率、带宽、PSRAM  
- **电源管理**：LDO/Buck/Boost，注意输入范围、电流、效率、热设计

---

## 输出检查清单
- 覆盖所有功能  
- 给出具体器件  
- 列出关键电气、接口、性能、资源  
- 标明来源和待确认项  
- 区分器件选型和SDK适配  
- 避免只选SDK已有器件  
- 标明供电、电平、资源、量产风险  
- 遵守SDK目录约束
