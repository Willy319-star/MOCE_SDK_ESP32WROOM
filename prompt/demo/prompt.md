请基于当前 SDK，结合 SDK 中已有的外设驱动和参考例程，帮我进行一个“多传感器融合 + 双电机 PID 控制”的桌面伴侣工程功能分析。


本工程包含以下外设：

* TOF 测距模块：用于检测前方距离，判断是否有障碍物、是否靠近用户、是否需要减速或避障。
* MPU6050：用于检测设备姿态，判断是否倾斜、被拿起、发生碰撞或剧烈晃动。
* TW-TTS 语音模组：用于输出语音反馈，例如开机提示、互动提示、避障提示、异常提示等。
* 双电机与编码器：用于实现桌面伴侣移动。
* OLED屏幕：打印TOF和MPU6050的数据，以及当前状态机状态。

请分析该工程的主要功能设计：

1. 系统初始化
   上电后完成传感器、电机、编码器、OLED和语音模组的初始化。初始化过程中电机保持停止，完成后进入自检。

2. 传感器采集与融合
   系统需要周期性采集 TOF 距离、MPU6050 姿态、电机编码器数据，并综合判断当前状态。
   TOF 负责距离和障碍判断，MPU6050 负责姿态安全判断，编码器负责运动反馈和电机状态判断。
   三者融合后决定设备是否可以移动、是否需要减速、避障、停止或进入保护状态。

3. 双电机运动控制
   系统可以通过板上按键控制电机正转、反转；

4. 语音交互
   TW-TTS 模组根据系统状态进行语音播报，例如：
   开机完成、进入待机、检测到用户靠近、前方有障碍、正在避障、设备被拿起、姿态异常、电机异常等。
   语音提示应由状态变化触发，避免在循环中重复播报。

5. 屏幕交互
    OLED上显示数据以及当前状态机状态。

系统运行状态机逻辑：

* INIT 初始化状态：系统上电后进入，完成外设初始化，成功后进入自检，失败则进入故障状态。
* SELF_CHECK 自检状态：检查 TOF、MPU6050、编码器、电机和 TTS 是否正常，通过后进入待机。
* IDLE 待机状态：设备静止等待，持续检测距离和姿态。如果检测到用户靠近，进入互动状态；如果需要移动，进入移动状态；如果异常，进入保护状态。
* INTERACT 互动状态：当用户靠近且设备姿态稳定时，通过 TTS 播放互动语音，可根据需要进入移动或返回待机。
* MOVE 移动状态：双电机运行
* AVOID 避障状态：当前方障碍物过近时，设备减速、停止或转向避障。避障成功后返回移动或待机，失败则进入保护或故障状态。
* PROTECT 保护状态：当设备被拿起、倾斜过大、剧烈晃动、电机堵转或传感器异常时，立即停止电机并语音提示。异常解除后返回待机。
* ERROR 故障状态：当系统出现无法自动恢复的问题时进入，电机保持停止，等待用户复位或重新上电。
